# EasNote 权限使用说明

本文档详细说明EasNote应用所申请的权限及其用途，确保用户了解应用如何使用这些权限来提供服务。

## 保留的必要权限

### 1. 媒体访问权限（Android 13+）
- **权限名称**：
  - `READ_MEDIA_IMAGES` (读取媒体图片)
  - `READ_MEDIA_VIDEO` (读取媒体视频)
- **用途**：
  - 从设备相册选择图片添加到笔记
  - 未来版本支持从设备相册选择视频添加到笔记
  - 仅访问用户明确选择的媒体文件
- **必要性**：
  - 增强用户体验的重要功能，用于丰富笔记内容
  - Android 13+推荐的细粒度媒体权限

### 2. 相机权限
- **权限名称**：
  - `CAMERA` (相机)
  - `android.hardware.camera` (相机硬件支持)
  - `android.hardware.camera.autofocus` (相机自动对焦)
- **用途**：
  - 允许用户在笔记中直接拍摄照片并插入
  - 提供照片自动对焦功能，确保拍摄的图片清晰
- **必要性**：
  - 增强用户体验的重要功能，用于丰富笔记内容
  - 仅在用户主动使用相机功能时才会访问

### 3. 闪光灯权限
- **权限名称**：
  - `FLASHLIGHT` (闪光灯)
- **用途**：
  - 在光线不足的环境下，配合相机提供照明
  - 确保拍摄的图片在暗环境下也能清晰可见
- **必要性**：
  - 相机功能的辅助权限，提升拍摄体验
  - 仅在用户使用相机且开启闪光灯时才会激活

### 4. 网络状态权限
- **权限名称**：
  - `ACCESS_NETWORK_STATE` (访问网络状态)
  - `ACCESS_WIFI_STATE` (访问WiFi状态)
- **用途**：
  - 检测设备的网络连接状态
  - 为网络监控功能提供数据支持
  - 根据网络状态优化应用性能
- **必要性**：
  - 用于网络监控功能，帮助用户了解设备网络情况
  - 不用于发送或接收用户数据，仅用于状态检测

## 已移除的不必要权限

以下权限已从应用中移除，因为它们与EasNote的核心功能无关：

- `CHANGE_NETWORK_STATE` (修改网络状态)
- `MOUNT_UNMOUNT_FILESYSTEMS` (挂载/卸载文件系统)
- `VIBRATE` (振动)
- `READ_LOGS` (读取系统日志)
- `GET_ACCOUNTS` (获取账户信息)
- `READ_PHONE_STATE` (读取电话状态)
- `CHANGE_WIFI_STATE` (修改WiFi状态)
- `WAKE_LOCK` (唤醒锁)
- `WRITE_SETTINGS` (修改系统设置)

## 权限使用原则

EasNote遵循以下权限使用原则：

1. **最小权限原则**：仅申请实现核心功能所必需的权限
2. **透明使用**：清晰告知用户每个权限的具体用途
3. **用户控制**：允许用户在系统设置中管理和撤销权限
4. **安全使用**：不会将权限用于非声明的用途或与第三方共享

## 更新说明

本权限说明文档会随着应用功能的更新而及时调整。如果应用增加了新功能需要申请额外权限，我们会在更新时明确告知用户新权限的用途。
